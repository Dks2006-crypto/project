    GitHub - это

    это облачная платформа для хостинга IT-проектов и совместной разработки, под капотом которой находится популярная система контроля версий Git, а также полноценная социальная сеть для разработчиков.

    GitBush - это

    это интерфейс командной строки (CLI), который объединяет функциональность Git и Bash, популярной оболочки Unix.Он предлагает удобный способ взаимодействия с репозиториями Git, выполнения задач контроля версий и навигации по файловой системе.

    Регистрация на github

    Для того чтобы зарегистрироваться на GitHub нужно:

    1. Откройте страницу https://github.com в веб-браузере.

    2. Нажмите кнопку «Зарегистрироваться».

    3. Введите адрес электронной почты.

    4. Создайте пароль для новой учетной записи GitHub и введите имя пользователя.

    5. Выберите, хотите ли вы получать обновления и объявления по электронной почте, а затем нажмите кнопку «Продолжить».

    6. Проверьте свою учетную запись, решив головоломку.

    7. После проверки учетной записи нажмите кнопку «Создать учетную запись».

    8. GitHub отправит код запуска по адресу электронной почты. Введите код запуска в диалоговом окне «Ввод кода», а затем нажмите клавишу «ВВОД».

    Отправка проекта в GitHub

    Для того чтобы отправить проект в GitHub нужно:

    1. Создайть новую репозиторию для своего проекта

    2. Загрузить файлы в репозиторий вашего проекта

    3. Отредактировать файл README для репозитория вашего проекта

    Скачивание файлов из GitHub

    GitHub.com является домом для миллионов проектов программного обеспечения с открытым исходным кодом, которые можно копировать, настраивать и использовать для собственных целей. Существуют различные способы получения копии файлов репозитория на GitHub. Вы можете: Скачайте моментальный снимок файлов репозитория в виде ZIP-файла на собственный (локальный) компьютер. Клонируйте репозиторий на локальный компьютер с помощью Git. Вилку репозитория для создания нового репозитория на GitHub. Каждый из этих методов имеет собственный вариант использования, который мы объясним в следующем разделе. В этом руководстве рассматривается скачивание файлов репозитория на локальный компьютер. Например, если вы нашли интересное содержимое в репозитории на GitHub, скачивание — это простой способ получить копию содержимого без использования Git или применения управления версиями.

    Удаление учетной записи

    При удалении личной учетной записи удаляются все репозитории, вилки частных репозиториев, вики-сайтов, проблемы, запросы на вытягивание и страницы, принадлежащие вашей учетной записи. Созданные вами проблемы и запросы на вытягивание и комментарии, которые вы оставили в репозиториях других пользователей, не будут удалены. Ваши ресурсы и комментарии будут связаны с пользователем-призраком.

    Руководство по репозиториям

    В репозиториях GitHub хранятся различные проекты. В этом руководстве вы создадите репозиторий и зафиксируете первое изменение.

    1. В правом верхнем углу любой страницы выберите , а затем нажмите кнопку Создать репозиторий.

    2. Введите короткое, запоминающееся имя репозитория. Например, "hello-world".

    3. При необходимости добавьте описание репозитория. Например, «Мой первый репозиторий на GitHub».

    4. Выберите видимость репозитория. Дополнительные сведения см. в разделе Сведения о репозиториях.

    5. Выберите Инициализировать этот репозиторий с помощью файла сведений.

    6. Нажмите кнопку Создать репозиторий.
